URL 過濾技術全解析：企業如何有效管控 Web 存取風險

超過 90% 的惡意軟體透過 Web 流量傳播。URL 過濾是企業安全的第一道防線，但技術選型與策略設計同樣關鍵。

Hexion Networks 資安研究團隊·2025 年 8 月 18 日·6 分鐘閱讀

為何 URL 過濾是企業必備的安全基礎

根據 Verizon 資料外洩調查報告（DBIR），Web 流量持續是惡意軟體散播的主要管道之一，超過 90% 的惡意軟體透過 Web 連線傳遞。釣魚攻擊（Phishing）、Drive-by 下載、惡意廣告（Malvertising）以及 C2（Command and Control）通訊，都仰賴受害者的瀏覽器發出 HTTP/HTTPS 請求。

URL 過濾在安全架構中的位置，使其能夠在最早期就攔截威脅：在用戶的瀏覽器嘗試連接惡意網站之前，防火牆就能檢查該 URL 並決定是否允許連線。這比等待惡意程式碼下載到端點再由防毒軟體偵測，提前了整個攻擊鏈的好幾個步驟。

URL 過濾同時也是生產力管理的工具：企業可封鎖社群媒體、線上遊戲、購物網站等，確保員工的頻寬與工作時間用於業務目的。這在金融服務業、政府機關等高監管環境中，往往也是法規合規的必要控制。

技術比較：DNS 過濾、代理過濾與 NGFW 整合

DNS 層過濾
部署最簡單

在 DNS 解析層攔截對惡意域名的查詢。優點是部署極為簡單（只需更改 DNS 伺服器設定），且能覆蓋所有裝置（包括 IoT）。缺點是只能針對整個域名，無法精準控制同一域名下的特定子頁面或路徑。此外，攻擊者可使用 DNS over HTTPS（DoH）繞過。

+ 覆蓋所有設備 + 極低延遲 - 無法做 URL 路徑層控制 - DoH 可繞過
代理伺服器過濾（Proxy-based）
傳統企業方案

所有 Web 流量強制通過代理伺服器，可做到 URL 路徑層的精細控制與完整日誌記錄。缺點是所有設備需要配置代理設定，成為單一故障點，且對效能有一定影響。雲端 SaaS 應用的普及使傳統 Proxy 的適用性下降。

+ URL 路徑精細控制 + 完整日誌 - 設備需個別配置 - 單一故障點
NGFW 整合 URL 過濾
Hexion 推薦方案

將 URL 過濾整合在 NGFW 引擎中，結合 DPI 的應用識別能力，在網關處對所有流量進行統一的 URL 分類與策略執行。無需額外設備或端點配置，與 IPS、惡意軟體防護、SSL 解密緊密整合，提供最全面的防護。

+ 無需端點設定 + 與 IPS/AV 整合 + SSL 解密配合 - 需足夠的硬體效能

URL 分類資料庫的重要性：Lionfilter 億級 DB

URL 過濾系統的核心競爭力在於分類資料庫的品質與規模。一個資料庫如果只涵蓋 1,000 萬個 URL，面對每天新增的數百萬個新域名（包括惡意域名），其防護效果將大打折扣。

LIONIC 的 Lionfilter URL 資料庫具備以下優勢：

  • 億級 URL 收錄量：涵蓋超過 10 億條 URL 記錄，是市場上規模最大的分類資料庫之一，確保即使是長尾的小眾網站也有分類記錄
  • 200+ 精細分類：遠超傳統的 50-80 種分類，能精確區分「社群媒體-Facebook」與「社群媒體-LinkedIn」等細分類別，讓企業能制定更細緻的政策
  • 亞太區本地化：針對台灣、日本、韓國、中國、東南亞的本地內容進行特別分類，避免因語言障礙造成的誤分類
  • 即時更新：雲端分類引擎能在數分鐘內對新出現的威脅性 URL 完成分類，不必等待離線資料庫更新

HTTPS 解密的必要性與挑戰

若不進行 SSL/TLS 解密，URL 過濾在 HTTPS 流量上只能看到主機名稱（SNI 欄位），無法看到完整的 URL 路徑。這意味著：攻擊者只需使用 HTTPS，就能讓 URL 過濾僅能做到域名層級的控制，而無法阻擋來自合法域名下特定頁面的威脅。

例如，Google Drive（drive.google.com）是業務工具，不應整個封鎖，但其中可能有人分享惡意 PDF。若無 HTTPS 解密，防火牆無法阻擋對特定惡意檔案 URL 的存取。

但 HTTPS 解密帶來隱私與法規考量。台灣《個人資料保護法》要求企業對員工個資的處理有明確法律依據。Hexion 建議企業在部署前：制定並公告員工 BYOD 及 IT 使用政策、建立解密範圍白名單排除金融、健康等敏感服務、確保解密後的流量日誌受嚴格存取控制保護。

企業 URL 過濾策略設計

有效的 URL 過濾策略不是「越嚴格越好」，過度限制會阻礙員工生產力，引發繞過行為（使用熱點等）。策略設計應基於業務需求與風險評估：

BLOCK ALL
強制封鎖類別

惡意軟體、釣魚網站、間諜軟體、匿名代理、暗網、成人內容、賭博

LOG ONLY
監控記錄類別

社群媒體、串流媒體、雲端儲存（記錄但允許，分析異常上傳行為）

ALLOW
明確允許類別

商業網站、資訊科技、教育、新聞、業務必要的 SaaS 應用

合規與日誌管理

URL 過濾日誌不只是資安工具，也是法規合規的重要依據。對於受監管行業（金融、醫療、政府），詳細的 Web 存取記錄往往是稽核必要項目。

URL 過濾日誌應包含的關鍵欄位：

  • 時間戳記（精確到毫秒）
  • 來源 IP / 使用者名稱（整合 AD 認證）
  • 完整 URL（含路徑，需要 SSL 解密）
  • URL 分類與子分類
  • 執行動作（ALLOW / BLOCK / REDIRECT）
  • 資料傳輸量（上傳/下載位元組數）

日誌保存期限建議至少 6 個月（線上），1 年（歸檔）。金融業通常需保存 5-7 年。應將 URL 過濾日誌整合至 SIEM，設定告警規則偵測大量存取封鎖、異常的外部資料傳輸等行為。
