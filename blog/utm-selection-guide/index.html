NGFW 選購指南：中小企業到大型企業如何挑選適合的次世代防火牆

市場上 NGFW 產品琳瑯滿目，規格數字令人眼花繚亂。本指南提供實務選型框架，幫助台灣企業找到最適合的防火牆解決方案。

Hexion Networks 資安研究團隊·2025 年 8 月 5 日·8 分鐘閱讀

NGFW vs 傳統防火牆：核心差異

傳統防火牆（Stateful Firewall）根據 IP 位址、連接埠號碼和傳輸協議（TCP/UDP）做出允許或拒絕決策。這種「Layer 3/4 防火牆」在 1990 年代是業界標準，但面對現代威脅，它有三個根本性的侷限：

  • 無法識別應用程式——攻擊流量可偽裝成 HTTP（port 80）繞過規則
  • 無法檢查加密流量——HTTPS 對傳統防火牆完全透明
  • 無法識別使用者——規則基於 IP，無法對應到具體員工

次世代防火牆（NGFW）在這三個維度上全面超越：

APPLICATION AWARENESS

識別 1,400+ 應用程式，無論使用哪個連接埠

SSL INSPECTION

解密並檢查 HTTPS 流量，不讓加密成為盲點

USER IDENTITY

整合 AD/LDAP，政策可對應到使用者而非 IP

選型前的自我評估：五個關鍵問題

在比較規格之前，先回答這五個問題，能幫助你釐清實際需求：

  1. 企業規模與用戶數？50 人以下的小型辦公室與 300 人的中型企業，在並發連線需求上有天壤之別。選型規格必須對應實際使用規模，不是越大越好。
  2. 有沒有 OT/IoT 環境需求？純 IT 環境與有工廠機台的製造業，對防火牆的功能需求截然不同。OT 環境需要工業協議 DPI 支援。
  3. IT 團隊的技術能力？功能強大的防火牆若無人能運用，不如選擇介面簡單、有本地支援的方案。
  4. 是否有法規合規需求？金融業需要詳細的稽核日誌；醫療業需要 HL7/DICOM 流量保護；政府機關可能有特定的採購認證要求。
  5. 預算與 TCO 考量？除設備購置成本外，也需考慮授權費、維護費、訓練費、以及未來擴充成本。

關鍵規格解讀：別被數字欺騙

廠商規格表中最容易誤導買家的三個數字：

防火牆吞吐量 vs 防護吞吐量（Threat Protection Throughput）

廠商標示的「10 Gbps 防火牆吞吐量」通常是在只啟用 L3/L4 封包過濾、不啟用任何安全功能的條件下測得。啟用 DPI、IPS、URL 過濾、SSL 解密後，實際吞吐量可能只剩 30-50%。務必要求廠商提供啟用全功能後的「防護吞吐量」數字，並要求說明測試條件。

最大並發連線數（Maximum Concurrent Sessions）

這個數字決定防火牆能同時追蹤多少個連線狀態。對於員工人數少但應用程式多（如視訊會議、SaaS 應用）的環境，並發連線需求可能遠高於預期。一般估算是每個用戶 500-2,000 個並發連線。

每秒新建連線率（New Sessions per Second）

在 DDoS 攻擊或突發流量情境下，防火牆建立新連線的速度決定了它能否在高負載下維持服務。對於 Web 服務器保護場景，這個指標尤為重要。低速率的防火牆在攻擊時可能成為瓶頸。

OT 環境的特殊防火牆需求

若企業有工廠、機台或工業控制系統環境，標準的 IT 防火牆可能無法滿足需求。OT 防火牆的選型需額外考慮：

  • 工業協議支援：需能識別並控制 Modbus、DNP3、EtherNet/IP、PROFINET、OPC-UA 等工業協議
  • 硬體環境規格：工廠環境溫度範圍大（-40°C 至 70°C），震動、灰塵問題嚴重，應選用工業級規格的硬體，無風扇設計尤為重要
  • 被動部署支援：在不中斷生產的前提下部署，需要支援 TAP/SPAN 鏡像模式，先監控後執行
  • 低延遲保證：某些即時控制系統對延遲極為敏感，防火牆的介入必須在微秒級

TCO 計算與常見採購陷阱

總持有成本（TCO）計算是防火牆選型最常被忽視的環節。採購時應將以下成本納入 3-5 年期計算：

硬體成本

設備購置費、HA 備援機、安裝佈線

授權費用

年度訂閱（IPS特徵、URL資料庫、威脅情報）往往是設備費的 20-30%/年

維護與支援

原廠保固、硬體維護合約、技術支援（本地中文支援尤為重要）

人力成本

初始設定、日常管理、政策調整、事件回應所需的 IT 人力

常見陷阱：部分廠商以低廉的硬體費吸引買家，但授權費極高；另有廠商不提供本地中文支援，導致部署問題難以解決。Hexion 提供完整的台灣在地服務，包括繁體中文介面、本地技術支援與教育訓練。
